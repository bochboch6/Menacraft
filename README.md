# 🔍 MenaCraft — Content Authenticity & Trust Verification Platform

> Hackathon Project — Fighting the Trust Crisis in the Digital Age

---

## 📌 Problem Statement

In today's digital age, anyone can create content that looks real but isn't. Photos, videos, documents, and social media posts can be subtly altered, AI-generated, or entirely fabricated. Citizens, institutions, and businesses can no longer rely on what they see online.

**MenaCraft** is a multi-axis verification platform that detects AI-generated content, checks contextual consistency, and assesses source credibility — all in one unified pipeline.

## 🧩 The Three Axes

---

### 🤖 Axis 1 — Content Authenticity (AI Detection)

**Goal:** Detect whether any type of content was generated by AI or created by a human.

#### Supported Content Types

| Type | Tool / API | What is detected |
|---|---|---|
| Text / PDF | Copyleaks API | AI-generated text vs human writing |
| Image | Hive AI API | AI-generated image, deepfake detection |
| Audio | Hive AI API | AI-generated / synthetic voice detection |
| Video | Hive AI API | Deepfake video, AI-generated video detection |

#### How it works

```
Content uploaded (text / PDF / image / audio / video)
              │
    ┌─────────┴─────────┐
    │   Type detection   │
    └─────────┬─────────┘
              │
   ┌──────────┼──────────┐
   ▼          ▼          ▼
 Text/PDF   Image      Audio/Video
   │          │          │
Copyleaks  Hive AI    Hive AI
   │          │          │
   └──────────┴──────────┘
              │
    Score aggregation
              │
   Verdict + Confidence + Reasons
```

#### Text/PDF Output Example
```json
{
  "verdict":     "ai",
  "aiScore":     94.0,
  "humanScore":  6.0,
  "reasons": [
    "Sentence structure is uniform and highly predictable",
    "Word choices are statistically typical of AI generation",
    "Lack of personal tone or natural human inconsistencies"
  ]
}
```

#### Image/Video Output Example
```json
{
  "verdict": {
    "ai_generated": "AI_GENERATED",
    "deepfake":     "NO_DEEPFAKE"
  },
  "scores": {
    "ai_generated": 0.98,
    "deepfake":     0.03
  },
  "generators": [
    { "name": "midjourney", "score": 0.87 }
  ]
}
```

#### Technologies
- **Copyleaks API** — AI text detection (synchronous, per-sentence classification)
- **Hive AI API** — image/audio/video deepfake and AI generation detection
- **PyMuPDF** — PDF text extraction
- **Python FastAPI** — microservice wrapping Hive + Copyleaks
- **Node.js / Express** — API gateway

### 🖼️ Axis 2 — Contextual Consistency (Caption Alignment)

**Goal:** Detect whether a caption or claim accurately describes the visual content it accompanies.

#### How it works

```
User submits: Image + their own caption/description
              │
    ┌─────────┴──────────┐
    │  Gemini Vision AI  │  ← AI describes the image automatically
    │  (extractor.py)    │
    └─────────┬──────────┘
              │
    AI-generated description
              │
    ┌─────────┴──────────┐
    │   Cohere Embed     │  ← Semantic similarity scoring
    │  embed-english-v3  │
    └─────────┬──────────┘
              │
    Similarity score (0–100%)
              │
    Verdict + Explanation
```

#### Input
```json
{
  "description": {
    "scene_overview":    "Outdoor protest in a city square",
    "people":            "Large crowd holding signs",
    "visible_text":      ["Freedom", "Justice"],
    "location_clues":    ["European architecture"],
    "full_description":  "A large crowd gathered in a city square..."
  },
  "caption": "Peaceful celebration in a European capital"
}
```

#### Output Example
```
Alignment Score : 34%
Verdict         : ⚠️ LOW ALIGNMENT
Explanation     : The caption describes a celebration but the image
                  shows signs of protest. Key elements do not match.
```

#### Technologies
- **Gemini Vision AI** — automatic image description (scene, people, text, location)
- **Cohere** `embed-english-v3.0` — semantic similarity between caption and description
- **Python** — analysis backend (`analyzer.py`)

### 🌐 Axis 3 — Source Credibility

**Goal:** Assess whether the source of content can be trusted using three levels of analysis.

---

#### 3A — URL / Domain Credibility Test

Analyzes a URL or plain text for credibility signals.

**Signals checked:**

| Signal | Risk Added |
|---|---|
| Suspicious TLD (.xyz, .tk, .cf...) | +30 |
| Domain name too long or random | +15–20 |
| Typosquatting of trusted sources (bbc, cnn...) | +40 |
| Domain age < 30 days (WHOIS) | +40 |
| HTTP instead of HTTPS | +15 |
| URL shortener (bit.ly, t.co...) | +25 |
| ALL CAPS clickbait words | +20 |
| Excessive punctuation (!!! ???) | +15 |
| Sensational keywords (BREAKING, EXPOSED...) | +35 |
| No source attribution | +15 |

**Input:** `{ "input": "https://suspicious-news123.xyz/article" }`

**Output:**
```json
{
  "riskScore": 75,
  "riskLabel": "🔴 HIGH RISK",
  "flags": [
    "⚠️  Suspicious domain extension detected",
    "🚨 Domain created only 12 days ago",
    "✅  URL uses HTTPS (secure)"
  ],
  "reasons": [
    "Multiple credibility signals failed simultaneously",
    "Source cannot be independently verified"
  ]
}
```

---

#### 3B — Social Media Account Behavior Test

Analyzes Facebook and Instagram account behavior using signals from MediaExtractor.

**Supported platforms:** Facebook, Instagram

**Signals checked:**

| Signal | Risk Added |
|---|---|
| No images found | +20 |
| Very few images (< 3) | +10 |
| Zero engagement (likes/comments/shares) | +25 |
| Very low engagement ratio (< 1%) | +15 |
| Post content < 10 words | +15 |
| Sensational keywords in caption | +25 |
| Shortened URL in post | +20 |

**Input:** MediaExtractor result object (Facebook or Instagram)

**Output:**
```json
{
  "platform":    "instagram",
  "riskLabel":   "🔴 HIGH RISK",
  "riskScore":   80,
  "description": "This account shows multiple suspicious behavioral patterns.",
  "flags": [
    "⚠️  No images found on the account",
    "⚠️  Zero engagement detected",
    "⚠️  Sensational keywords: BREAKING, SHARE NOW"
  ],
  "reasons": [
    "Engagement pattern inconsistent with follower count",
    "Content uses emotional manipulation tactics"
  ]
}
```

---

#### 3C — Full Pipeline Test (Any URL)

Combines all axes into one unified analysis for any URL.

```
Any URL submitted
        │
        ▼
MediaExtractor.extract(url)
        │
        ├──────────────────────────────────┐
        ▼                                  ▼
Axis 1: AI Detection              Axis 3A: URL Credibility
(text → Copyleaks)                (domain → WHOIS checks)
(images → Hive AI)
(videos → Hive AI)
        │                                  │
        ▼                                  ▼
Axis 2: Caption Alignment         Axis 3B: Account Behavior
(image + caption → Cohere)        (engagement + content signals)
        │                                  │
        └──────────────┬───────────────────┘
                       ▼
              Combined Final Report
         (AI score + Alignment + Risk level)
```

## 🚀 Getting Started

### 1 — Install dependencies

```bash
npm install
pip install -r requirements.txt
```

### 2 — Set environment variables

```bash
export COPYLEAKS_EMAIL="your_email"
export COPYLEAKS_API_KEY="your_key"
export HIVE_API_KEY="your_key"
export COHERE_API_KEY="your_key"
export APIFY_TOKEN="your_token"
```

### 3 — Start all services

```bash
# Terminal 1 — Python extractor
cd extractor-service && npm run start:python

# Terminal 2 — Node extractor wrapper
cd extractor-service && npm start

# Terminal 3 — Hive detector service
cd detector-service && HIVE_API_KEY="your_key" npm start

# Terminal 4 — Central server
node app.js
```

## 🎯 Use Cases

**Use Case 1 — Journalist verifying an article**
A journalist uploads a PDF press release → system extracts text → Copyleaks detects 94% AI-generated → journalist alerted before publishing.

**Use Case 2 — Citizen checking a viral post**
A user pastes a Facebook post URL → system checks zero engagement + sensational keywords → returns HIGH RISK with reasons.

**Use Case 3 — Fact-checker verifying a news image**
A fact-checker uploads an image with its caption → Gemini describes the image → Cohere measures alignment → 34% match → caption flagged as misleading.

**Use Case 4 — Editor verifying a suspicious domain**
An editor submits a URL → WHOIS shows domain created 3 days ago + suspicious TLD → HIGH RISK returned immediately.

**Use Case 5 — Full pipeline analysis**
Any URL submitted → MediaExtractor extracts all content → Axis 1 detects AI in text and images → Axis 2 checks caption alignment → Axis 3 checks domain and account behavior → unified report returned.

## ⚠️ Limitations

- AI text detection supports **English only** (multilingual planned)
- Account analysis requires **Apify token** for Instagram/Facebook scraping
- Hive AI free tier: **20 requests/day**
- Caption alignment works best with **descriptive captions** (short captions may score low)
- Style analysis requires **minimum 6 posts** for reliable clustering
